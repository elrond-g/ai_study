# software-design-philosophy-skill

> https://github.com/luoling8192/software-design-philosophy-skill

## 项目简介

software-design-philosophy-skill 是一套针对后端和软件架构的 Claude Code Skill，将经典软件设计哲学（SOLID、DRY、KISS 等）和现代工程实践编码为 AI 可执行的规范检查和重构指导。

## 核心功能

- **架构审查**：检查代码是否违反 SOLID 原则
- **重构建议**：基于设计模式给出具体的重构方案
- **代码异味检测**：识别过度耦合、职责不清等问题
- **API 设计规范**：REST API、接口设计的最佳实践检查

## 覆盖的设计原则

- **SOLID**：单一职责、开闭原则、里氏替换、接口隔离、依赖倒置
- **DRY**：避免重复，提取抽象
- **KISS**：保持简单，抵制过度设计
- **YAGNI**：不实现当前不需要的功能
- **设计模式**：工厂、观察者、策略等 GoF 模式的适当使用

## 使用场景

- Code Review 时快速识别设计问题
- 重构遗留代码前的评估
- 培养团队的软件设计意识
